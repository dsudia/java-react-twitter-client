Fitter Docs

What is Fitter?

Fitter is a Tweet filtering service that can provide several analyses of Tweets.

When you make a request to Fitter, you can search tweets by keyword, and get back Kincaid Readability Scores and Sentiment Analysis of the tweets.

Acquiring an API Key

To get an API key, register by clicking Sign Up on the navbar above. When you register, an API key is automatically generated for you.

To see your API key, please login using the username and password you chose when registering

NOTE: Login is not currently working, please email dsudia@gmail.com with your username to receive your API key.

Sending Requests

Clients connect to Fitter through a websocket. When clients connect, they must send through a JSON object with the following keys:

  1. apiKey: your key, in string form
  2. keywords: any keywords you want to filter tweets by (in text body), in array form
  3. semantic: a boolean indicating whether or not you want sentiment scores returned
  4. readability: a boolean indicating whether or not you want readability scores returned

Socket connection requests should be made to "/tweets". The subscription url is "/tweets/{yourapikey}".

Processing Data

Fitter returns a JSON object that includes the following data:

  1. Tweet id
  2. Tweet text
  3. Tweet author
  4. Author's profile image url
  5. Type of attached media (if any)
  6. Attached media url (if any)
  7. Sentiment score of text (if requested)
  8. Readability score (if requested)

Sentiment analysis scores are valued as follows:

  • 0: Very Negative
  • 1: Negative
  • 2: Neutral
  • 3: Positive
  • 4: Very Positive
